# 智能问答接口前端调用逻辑文档

## 1. 接口概述

本文档描述了车辆数据智能问答系统的前端接口调用逻辑，系统基于 Spring Boot 3.2.1 开发，提供了一系列 RESTful API 用于实现智能问答功能。所有接口均需要通过 Sa-Token 认证后才能访问。

### 核心功能

- 创建问答会话
- 提交问题查询
- 重新生成答案
- 获取查询详情
- 获取会话历史
- 删除会话

## 2. 认证方式

系统使用 Sa-Token 进行身份认证，前端需要在请求头中携带有效的 token。

### 认证流程

1. 前端用户登录系统获取 token
2. 在后续的智能问答接口调用中，在请求头中添加 `Authorization: Bearer {token}`
3. 后端通过 `@SaCheckLogin` 注解验证 token 有效性

### 注意事项

- 所有智能问答相关接口均需要认证，未认证的请求会被拒绝
- token 过期后需要重新登录获取

## 3. 接口详细说明

### 3.1 创建问答会话

**接口地址**：`POST /api/v1/qa/session`

**功能描述**：创建新的智能问答会话，用于后续的问题查询。

**请求参数**：

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| userId | String | 是 | 用户ID |
| userName | String | 是 | 用户名 |

**请求示例**：

```json
{
  "userId": "1001",
  "userName": "测试用户"
}
```

**响应参数**：

| 参数名 | 类型 | 描述 |
|--------|------|------|
| sessionId | String | 会话ID |
| status | String | 会话状态 |
| createTime | LocalDateTime | 创建时间 |
| lastActivityTime | LocalDateTime | 最后活动时间 |

**响应示例**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "sessionId": "sess_1234567890",
    "status": "ACTIVE",
    "createTime": "2024-01-01T12:00:00",
    "lastActivityTime": "2024-01-01T12:00:00"
  }
}
```

### 3.2 提交问题查询

**接口地址**：`POST /api/v1/qa/query`

| templateId | Long | 否 | 使用的模板ID |
| parameters | Map<String, Object> | 否 | 参数映射 |
| userId | String | 否 | 用户ID |
| userName | String | 否 | 用户名 |

```json
{
```

## 4. 调用流程

### 4.1 典型调用流程

1. **创建会话**：用户进入智能问答页面时，调用 `POST /api/v1/qa/session` 创建会话
2. **提交问题**：用户输入问题后，调用 `POST /api/v1/qa/query` 提交查询
3. **展示结果**：接收并展示查询结果，包括答案、可视化数据等
4. **后续操作**：
   - 如果对答案不满意，调用 `POST /api/v1/qa/query/{queryId}/regenerate` 重新生成答案
   - 查看历史记录，调用 `GET /api/v1/qa/session/{sessionId}/history`
   - 查看查询详情，调用 `GET /api/v1/qa/query/{queryId}`
   - 结束会话，调用 `DELETE /api/v1/qa/session/{sessionId}`

### 4.2 会话管理

- 一个用户可以创建多个会话
- 会话用于组织相关的查询记录
- 建议在用户离开智能问答页面时删除会话，以节省服务器资源

## 5. 错误处理

### 5.1 常见错误类型

| 错误类型 | 错误码 | 描述 | 处理建议 |
|----------|--------|------|----------|
| NOT_LOGIN | 401 | 未登录或token无效 | 重新登录获取token |
| PARAM_ERROR | 400 | 参数错误 | 检查请求参数是否符合要求 |
| SESSION_NOT_FOUND | 404 | 会话不存在 | 重新创建会话 |
| QUERY_NOT_FOUND | 404 | 查询不存在 | 检查查询ID是否正确 |
| DATABASE_ERROR | 500 | 数据库错误 | 检查数据库连接和SQL语句 |
| AI_SERVICE_ERROR | 500 | AI服务错误 | 检查Tongyi Qianwen API配置 |
| SYSTEM_ERROR | 500 | 系统错误 | 联系管理员 |

### 5.2 错误响应格式

```json
{
  "code": 错误码,
  "message": "错误消息",
  "data": null
}
```

### 5.3 前端错误处理建议

1. **全局错误拦截**：在前端请求拦截器中统一处理错误
2. **token过期处理**：检测到401错误时，跳转到登录页面
3. **参数错误处理**：显示具体的参数错误信息给用户
4. **系统错误处理**：显示友好的错误提示，避免直接展示技术错误信息
5. **重试机制**：对于网络错误等临时性问题，可实现自动重试

## 6. 示例代码

### 6.1 Axios 示例

```javascript
// 1. 创建会话
async function createSession(userId, userName) {
  try {
    const response = await axios.post('/api/v1/qa/session', {
      userId,
      userName
    }, {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    return response.data.data;
  } catch (error) {
    handleError(error);
    throw error;
  }
}

// 2. 提交问题查询
async function submitQuery(sessionId, question, templateId = null, parameters = {}) {
  try {
    const response = await axios.post('/api/v1/qa/query', {
      sessionId,
      question,
      templateId,
      parameters,
      userId: localStorage.getItem('userId'),
      userName: localStorage.getItem('userName')
    }, {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    return response.data.data;
  } catch (error) {
    handleError(error);
    throw error;
  }
}

// 3. 重新生成答案
async function regenerateAnswer(queryId) {
  try {
    const response = await axios.post(`/api/v1/qa/query/${queryId}/regenerate`, {}, {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    return response.data.data;
  } catch (error) {
    handleError(error);
    throw error;
  }
}

// 4. 获取查询详情
async function getQueryDetail(queryId) {
  try {
    const response = await axios.get(`/api/v1/qa/query/${queryId}`, {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    return response.data.data;
  } catch (error) {
    handleError(error);
    throw error;
  }
}

// 5. 获取会话历史
async function getSessionHistory(sessionId, page = 1, size = 20) {
  try {
    const response = await axios.get(`/api/v1/qa/session/${sessionId}/history`, {
      params: { page, size },
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    return response.data.data;
  } catch (error) {
    handleError(error);
    throw error;
  }
}

// 6. 删除会话
async function deleteSession(sessionId) {
  try {
    const response = await axios.delete(`/api/v1/qa/session/${sessionId}`, {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    return response.data;
  } catch (error) {
    handleError(error);
    throw error;
  }
}

// 错误处理函数
function handleError(error) {
  if (error.response) {
    // 服务器返回错误
    const { code, message } = error.response.data;
    switch (code) {
      case 401:
        // 未登录或token无效，跳转到登录页
        window.location.href = '/login';
        break;
      case 400:
        // 参数错误
        alert(`参数错误: ${message}`);
        break;
      case 404:
        // 资源不存在
        alert(`资源不存在: ${message}`);
        break;
      default:
        // 其他错误
        alert(`系统错误: ${message}`);
    }
  } else if (error.request) {
    // 请求发送成功但没有收到响应
    alert('网络错误，请检查网络连接');
  } else {
    // 请求配置有误
    alert('请求错误: ' + error.message);
  }
}
```

### 6.2 Fetch API 示例

```javascript
// 创建会话
async function createSession(userId, userName) {
  try {
    const response = await fetch('/api/v1/qa/session', {
      method: 'POST',

        'Content-Type': 'application/json',
```

## 7. 性能优化建议

### 7.1 前端优化

1. **请求防抖**：用户输入问题时，添加防抖处理，避免频繁发送请求
2. **缓存策略**：对于相同问题的查询结果，可以在前端进行缓存
3. **异步处理**：使用异步请求，避免阻塞UI线程
4. **分页加载**：获取历史记录时使用分页，避免一次性加载过多数据
5. **预加载**：在用户可能进行的操作前，提前加载相关数据

### 7.2 交互优化

1. **加载状态**：在请求过程中显示加载状态，提升用户体验
2. **错误提示**：提供清晰、友好的错误提示
3. **进度反馈**：对于处理时间较长的请求，提供进度反馈
4. **自动保存**：自动保存用户的问题和会话状态，避免刷新页面后丢失
5. **快捷键支持**：支持回车提交问题等快捷键操作

## 8. 安全注意事项

1. **token 安全**：避免在前端存储 token 到本地存储以外的地方，建议使用 localStorage 或 sessionStorage
2. **参数验证**：前端应验证用户输入的参数，避免发送恶意请求
3. **请求频率限制**：避免短时间内发送大量请求，防止服务器过载
4. **敏感信息保护**：不在前端存储或显示敏感信息
5. **HTTPS 使用**：确保所有请求通过 HTTPS 发送，防止数据被窃取
