# 智能问答系统完整方案设计

##  项目概述

基于通义千问大模型，结合RAG（Retrieval-Augmented Generation）技术，为xai-vehicledata微服务构建一个类飞书知识库的智能问答系统。系统支持自然语言问答、结构化数据查询、多轮对话，并提供丰富的数据可视化展示。

##  核心需求

- **用户群体**：外部客户
- **使用频率**：日均150次查询，低并发
- **准确性要求**：100%准确
- **响应时间**：≤10秒（不超20秒）
- **功能特性**：
  - 多轮对话支持
  - 复杂查询支持
  - 可视化数据展示
  - 自定义问题模板
  - 查询审计日志

### 核心组件说明

#### 1. RAG引擎 (Retrieval-Augmented Generation)
- **功能**：实现检索增强生成，结合外部知识和数据库内容
- **核心流程**：
  1. 问题理解与意图识别
  2. 相关文档/数据检索
  3. 上下文增强
  4. 答案生成

#### 2. SQL生成器
- **功能**：将自然语言转换为精确的SQL查询
- **技术**：基于大模型的Text-to-SQL + 模板匹配
- **安全**：SQL注入防护 + 查询白名单

#### 3. 缓存管理器
- **功能**：多级缓存策略，提升响应速度
- **实现**：Redis + 本地缓存 + 向量缓存

#### 4. 可视化服务
- **功能**：智能选择合适的图表类型展示数据
- **支持类型**：表格、柱状图、饼图、线图、仪表盘

## 模型调用策略

### 核心策略

#### 1. 多阶段推理流程
```
用户问题 → 意图识别 → 检索增强 → SQL生成 → 结果验证 → 答案生成 → 可视化选择
```

#### 2. Prompt工程设计

##### 问题理解Prompt
```prompt
你是一个专业的汽车智能驾驶评估系统助手。请分析用户的问题，提取以下信息：

问题：{user_question}

请返回JSON格式：
{
  "intent": "query_data|ask_question|compare_data",
  "entities": ["entity1", "entity2"],
  "query_type": "single_table|multi_table|aggregation|comparison",
  "visualization_preference": "table|chart|mixed",
  "complexity": "simple|medium|complex"
}
```

##### SQL生成Prompt
```prompt
基于以下数据库schema和用户问题，生成准确的SQL查询：

数据库表结构：
{table_schemas}

用户问题：{user_question}
上下文信息：{context}

请生成：
1. SQL查询语句
2. 查询结果的字段说明
3. 数据验证规则

返回格式：
{
  "sql": "SELECT ...",
  "fields": [{"name": "field1", "type": "string", "description": "描述"}],
  "validation_rules": ["rule1", "rule2"]
}
```

##### 答案生成Prompt
```prompt
基于查询结果，为用户生成自然语言答案：

原始问题：{user_question}
查询结果：{query_result}
数据字段：{field_descriptions}

要求：
1. 答案准确、简洁、有用
2. 突出关键数据
3. 提供必要的上下文解释
4. 建议合适的可视化方式

返回格式：
{
  "answer": "答案内容",
  "key_points": ["要点1", "要点2"],
  "visualization_suggestion": "table/bar/pie等",
  "confidence": 0.95
}
```

### 模型选择策略

#### 主模型：通义千问
- **型号**：qwen-turbo / qwen-plus / qwen-max
- **适用场景**：
  - qwen-turbo：简单查询，成本优先
  - qwen-plus：复杂查询，平衡性能
  - qwen-max：超复杂查询，准确性优先

#### 备选模型：开源模型
- **Qwen-7B/14B**：本地部署，隐私保护
- **ChatGLM**：国产替代，保证服务连续性

### 调用优化策略

#### 1. 智能路由
```java
public String routeModel(String question, QueryContext context) {
```

##  缓存机制

### 多级缓存架构

#### 1. L1缓存：本地缓存 (Caffeine)
```java
@Configuration
public class CacheConfig {
    @Bean
    public Cache<String, QAQueryResult> qaCache() {
        return Caffeine.newBuilder()
            .maximumSize(1000)
            .expireAfterWrite(30, TimeUnit.MINUTES)
            .recordStats()
            .build();
    }
}
```

#### 2. L2缓存：Redis分布式缓存
```java
@Service
public class RedisCacheService {
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    public QAQueryResult get(String key) {
        return (QAQueryResult) redisTemplate.opsForValue().get("qa:query:" + key);
    }

    public void set(String key, QAQueryResult value, long timeoutSeconds) {
        redisTemplate.opsForValue().set("qa:query:" + key, value, timeoutSeconds, TimeUnit.SECONDS);
    }
}
```

#### 3. L3缓存：向量缓存
- **问题向量缓存**：相似问题快速匹配
- **答案向量缓存**：相似答案复用
- **文档向量缓存**：知识库检索加速

### 缓存策略

#### 1. 缓存键设计
```java
public String generateCacheKey(String question, String userId, Map<String, Object> params) {
    String baseKey = question + "|" + userId;
    if (params != null && !params.isEmpty()) {
        baseKey += "|" + params.toString();
    }
    return DigestUtils.md5Hex(baseKey);
}
```

#### 2. 缓存失效策略
- **时间失效**：30分钟 - 24小时（基于数据更新频率）
- **LRU淘汰**：最近最少使用
- **主动失效**：数据更新时清除相关缓存
- **热点数据**：永不过期 + 定期刷新

#### 3. 缓存预热
- **系统启动时**：加载热门问题缓存
- **定时任务**：每日预热高频查询
- **用户行为**：基于用户历史预判需求

### 缓存监控
- **命中率统计**：实时监控缓存效果
- **性能监控**：缓存操作耗时统计
- **容量监控**：缓存使用率告警

### 模板匹配算法

#### 1. 多层匹配策略
```java
public TemplateMatchResult matchTemplate(String question, List<QATemplate> templates) {
    // 1. 精确关键词匹配
    List<QATemplate> exactMatches = templates.stream()
        .filter(t -> containsAllKeywords(question, t.getKeywords()))
        .collect(Collectors.toList());

    if (!exactMatches.isEmpty()) {
        return new TemplateMatchResult(exactMatches.get(0), MatchType.EXACT);
    }

    // 2. 模糊匹配
    List<TemplateScore> fuzzyMatches = templates.stream()
        .map(t -> new TemplateScore(t, calculateSimilarity(question, t)))
        .filter(s -> s.getScore() > 0.6)
        .sorted((a, b) -> Double.compare(b.getScore(), a.getScore()))
        .collect(Collectors.toList());

    if (!fuzzyMatches.isEmpty()) {
        return new TemplateMatchResult(fuzzyMatches.get(0).getTemplate(), MatchType.FUZZY);
    }

    // 3. 正则匹配
    for (QATemplate template : templates) {
        if (template.getQuestionPattern() != null &&
            Pattern.matches(template.getQuestionPattern(), question)) {
            return new TemplateMatchResult(template, MatchType.REGEX);
        }
    }

    return new TemplateMatchResult(null, MatchType.NONE);
}
```

#### 2. 相似度计算
```java
public double calculateSimilarity(String question, QATemplate template) {
    // 关键词相似度
    double keywordSimilarity = calculateKeywordSimilarity(question, template.getKeywords());

    // 语义相似度 (基于向量余弦相似度)
    double semanticSimilarity = calculateSemanticSimilarity(question, template.getDescription());

    // 权重组合
    return 0.7 * keywordSimilarity + 0.3 * semanticSimilarity;
}
```

```java
public ChartRecommendation recommendChart(DataFrame data, QueryIntent intent) {
    // 分析数据特征
    DataCharacteristics characteristics = analyzeDataCharacteristics(data);

    // 根据查询意图推荐
    List<ChartType> candidates = new ArrayList<>();

    switch (intent.getType()) {
        case COMPARISON:
            if (characteristics.getDimensions() <= 2) {
                candidates.add(ChartType.BAR);
                candidates.add(ChartType.RADAR);
            } else {
                candidates.add(ChartType.TABLE);
            }
            break;

        case TREND:
            candidates.add(ChartType.LINE);
            candidates.add(ChartType.AREA);
            break;

        case DISTRIBUTION:
            candidates.add(ChartType.PIE);
            candidates.add(ChartType.BAR);
            break;

        default:
            candidates.add(ChartType.TABLE);
            break;
    }

    // 返回最优推荐
    return new ChartRecommendation(candidates.get(0), 0.85, "基于数据特征和查询意图推荐");
}
```
