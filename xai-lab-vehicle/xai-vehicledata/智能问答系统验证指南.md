# 智能问答系统验证指南

## 📋 文档概述

本文档提供了完整的智能问答系统验证指南，涵盖功能验证、性能测试、错误处理等各个方面。通过本指南，你可以系统性地验证智能问答系统的所有核心功能。

## 🎯 验证目标

- ✅ 验证系统所有核心功能正常运行
- ✅ 验证数据同步、缓存、可视化等组件协同工作
- ✅ 验证系统性能满足业务需求
- ✅ 验证错误处理和容错能力
- ✅ 验证安全性和审计功能

## 🔧 环境要求

### 系统环境
- **操作系统**: Windows/Linux/macOS
- **Java版本**: JDK 17+
- **数据库**: MySQL 8.0+
- **缓存**: Redis 6.0+
- **注册中心**: Nacos 2.2+

### 数据准备
- ✅ `qa_question_template` 表包含6个预设模板
- ✅ `vehicle_domain_score` 表有测试数据
- ✅ `beeeval_open_case_score` 表有测试数据
- ✅ `qa_knowledge_base` 表包含车辆数据（通过同步接口导入）
- ✅ Redis服务正常运行
- ✅ 应用成功启动在端口8181
- ✅ 通义千问API Key已配置（在Nacos中）

### 工具准备
- ✅ cURL 或 Postman 用于API测试
- ✅ MySQL客户端用于数据验证
- ✅ Redis客户端用于缓存验证

#### 3.2 单车功能域详情
**验证步骤:**
```bash
curl -X POST http://localhost:8181/api/v1/qa/query \
  -H "Content-Type: application/json" \
  -d '{
    "sessionId": "sess-xxx-xxx-xxx",
    "question": "查询车辆ID为123的功能域评分详情",
    "userId": "test_user_001"
  }'
```

**预期SQL:**
```sql
SELECT * FROM vehicle_domain_score WHERE vehicle_id = 123 AND type = 1
```

**说明:**
- `vehicle_id` 对应 `vehicle_base_info.id`
- `type=1` 表示功能域总分，`domain_id` 关联 `vehicle_functional_domain.id`
- `type=2或3` 表示三级指标分，`domain_id` 关联 `vehicle_domain_index.id`

#### 3.3 多车对比分析
**验证步骤:**
```bash
curl -X POST http://localhost:8181/api/v1/qa/query \
  -H "Content-Type: application/json" \
  -d '{
    "sessionId": "sess-xxx-xxx-xxx",
    "question": "比较小米SU7和蔚来ET7的功能域评分",
    "userId": "test_user_001"
  }'
```

**预期SQL:**
```sql
SELECT CONCAT(b.brand, base.vehicle_model) AS vehicle_name, d.functional_domain_name, s.score
FROM vehicle_domain_score s
JOIN vehicle_base_info base ON s.vehicle_id = base.id
JOIN vehicle_brand b ON base.brand_id = b.id
JOIN vehicle_functional_domain d ON s.domain_id = d.id
WHERE s.vehicle_id IN (123, 456) AND s.type = 1
```

**说明:**
- 车辆名称通过 `vehicle_brand.brand` + `vehicle_base_info.vehicle_model` 拼接
- 功能域名称通过 `vehicle_functional_domain.functional_domain_name` 获取
- 系统会先从 `qa_knowledge_base` 中检索车辆信息，提取车辆ID

**预期SQL:**

JOIN vehicle_base_info base ON s.vehicle_id = base.id
JOIN vehicle_brand b ON base.brand_id = b.id
GROUP BY s.vehicle_id
