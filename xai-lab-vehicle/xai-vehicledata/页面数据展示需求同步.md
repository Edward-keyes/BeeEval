数据展示接口接入分析文档
📋 现有接口实现现状
通过分析现有代码，发现所有展示接口目前都使用硬编码数据或从原有业务表查询数据，并未接入我们同步的数据。
🔍 接口数据来源分析
1. queryScoreByBrandInfo - 功能域分数展示
当前实现：硬编码数据（车控域、出行域等6个功能域的分数）
数据来源：DomainTreeServiceImpl.queryScoreByBrandInfo()
接入点：可替换为从 vehicle_domain_score 表查询 type = 1 的功能域分数
2. bigModelCapabilityAssessmentBase - 认知能力和行动能力展示
当前实现：硬编码数据（14个基础能力指标）
数据来源：DomainIndexServiceImpl.largeModelCapabilityAssessment()
接入点：
认知能力：从 vehicle_domain_score 表查询 type = 3 且指标属于认知能力的记录
行动能力：从 vehicle_domain_score 表查询 type = 3 且指标属于行动能力的记录
3. queryIndexScore - 功能域下各指标分数展示
当前实现：从 vehicle_domain_index 和 vehicle_domain_tree 表查询
数据来源：DomainTreeServiceImpl.queryDomainIndexScore()
接入点：可替换为从 vehicle_domain_score 表查询 type = 2 的三级指标分数
4. multiple - 开源题目分值展示
当前实现：未知（文档中显示返回problemList）
接入点：从 beeeval_open_case_score 表查询指定车辆和指标的开源题目分数
5. baseDomainIndexScoreSort - 基础能力指标排序展示
当前实现：从多个表查询基础能力数据
接入点：从 vehicle_domain_score 表查询 type = 3 的所有基础能力分数
6. queryIndexScoreRank - 功能域指标排行榜
当前实现：基于queryIndexScore的扩展
接入点：同样使用 vehicle_domain_score 表数据
