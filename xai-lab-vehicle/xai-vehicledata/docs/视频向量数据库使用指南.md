# 视频向量数据库使用指南

## 一、系统概述

本系统实现了基于PGVector的视频向量存储和检索功能，支持：
- 批量向量化所有视频
- 增量向量化新视频
- 混合检索（向量 + 全文）
- 与智能问答系统集成

## 二、环境准备

### 1. 安装PGVector扩展

```sql
-- 连接PostgreSQL数据库
psql -U postgres -d your_database

-- 安装扩展
CREATE EXTENSION IF NOT EXISTS vector;
```

### 2. 执行数据库脚本

```bash
# 执行PGVector表结构创建脚本
psql -U postgres -d your_database -f src/main/resources/sql/pgvector_schema.sql
```

### 3. 配置文件

在 `application-qa.yml` 中已添加向量嵌入配置：

```yaml
qa:
  embedding:
    api-key: sk-86a6602e9ae64483afe0cdea91ab546a
    model: text-embedding-v2
    dimension: 1536
    batch-size: 20
    max-retries: 3
    retry-interval-ms: 1000
    request-timeout-seconds: 60
    enable-cache: true
    cache-expire-hours: 24
    similarity-threshold: 0.7
    top-k: 10
```

## 三、使用流程

### 步骤1：确保视频已分析

在向量化之前，视频必须先完成分析：

```bash
# 检查视频分析状态
curl -X GET http://localhost:8080/api/v1/video-analysis/status/{taskId}
```

### 步骤2：批量向量化所有视频

```bash
# 向量化所有未处理的视频
curl -X POST http://localhost:8080/api/v1/video-vectorization/batch \
  -H "Authorization: Bearer your_token" \
  -H "Content-Type: application/json" \
  -d '{}'

# 强制重新向量化所有视频
curl -X POST http://localhost:8080/api/v1/video-vectorization/batch \
  -H "Authorization: Bearer your_token" \
  -H "Content-Type: application/json" \
  -d '{
    "forceUpdate": true
  }'
```

### 步骤3：异步批量向量化（推荐）

```bash
# 异步处理，立即返回任务ID
curl -X POST http://localhost:8080/api/v1/video-vectorization/batch-async \
  -H "Authorization: Bearer your_token" \
  -H "Content-Type: application/json" \
  -d '{}'

# 使用返回的taskId查询进度
curl -X GET http://localhost:8080/api/v1/video-vectorization/status/{taskId} \
  -H "Authorization: Bearer your_token"
```

### 步骤4：增量向量化

```bash
# 向量化指定视频列表
curl -X POST http://localhost:8080/api/v1/video-vectorization/incremental \
  -H "Authorization: Bearer your_token" \
  -H "Content-Type: application/json" \
  -d '[1, 2, 3, 4, 5]'

# 向量化单个视频
curl -X POST http://localhost:8080/api/v1/video-vectorization/single/123 \
  -H "Authorization: Bearer your_token"
```

### 步骤5：检查向量化状态

```bash
# 检查单个视频是否已向量化
curl -X GET http://localhost:8080/api/v1/video-vectorization/check/123 \
  -H "Authorization: Bearer your_token"

# 获取未处理视频数量
curl -X GET http://localhost:8080/api/v1/video-vectorization/unprocessed-count \
  -H "Authorization: Bearer your_token"
```

## 四、智能问答集成

### 自动向量检索

系统已自动集成向量检索到智能问答流程中：

1. 用户提问时，系统自动判断是否需要视频检索
2. 如果需要，进行向量检索获取相关视频
3. 将视频分析结果作为上下文注入LLM
4. 生成包含视频分析内容的答案

### 示例问题

```
# 视频相关问题会自动触发向量检索
"车辆在智能驾驶功能域的视频表现如何？"
"分析车辆在车道保持测试中的视频表现"
"对比分析车辆在不同功能域的视频表现差异"
"查找车辆在应急处理方面的优秀案例视频"
```

## 五、API接口说明

### 1. 批量向量化接口

**POST** `/api/v1/video-vectorization/batch`

请求参数：
```json
{
  "videoIds": [1, 2, 3],  // 可选，为空则处理所有
  "forceUpdate": false,    // 是否强制更新
  "includeTranscript": true,
  "includeKeyframes": true
}
```

响应：
```json
{
  "taskId": "uuid",
  "totalVideos": 100,
  "processedCount": 100,
  "successCount": 98,
  "failedCount": 2,
  "status": "completed",
  "startTime": "2024-01-01T10:00:00",
  "endTime": "2024-01-01T10:30:00",
  "results": [...]
}
```

### 2. 异步批量向量化接口

**POST** `/api/v1/video-vectorization/batch-async`

立即返回任务ID，后台异步处理。

### 3. 增量向量化接口

**POST** `/api/v1/video-vectorization/incremental`

请求参数：
```json
[1, 2, 3, 4, 5]  // 视频ID列表
```

### 4. 单个视频向量化接口

**POST** `/api/v1/video-vectorization/single/{videoId}`

参数：
- `videoId`: 视频ID
- `forceUpdate`: 是否强制更新（默认false）

### 5. 查询处理状态接口

**GET** `/api/v1/video-vectorization/status/{taskId}`

### 6. 检查向量化状态接口

**GET** `/api/v1/video-vectorization/check/{videoId}`

### 7. 获取未处理数量接口

**GET** `/api/v1/video-vectorization/unprocessed-count`

## 六、向量检索原理

### 1. 向量化内容

系统对以下内容进行向量化：
- **视频摘要文本**: 视频类型、任务类型、描述、总结
- **转录文本**: 语音转文字内容
- **分析评分文本**: 综合评分、各维度评分、改进建议
- **关键帧描述**: 视觉描述内容

### 2. 检索流程

```
用户问题
    ↓
问题向量化 (通义千问Embedding API)
    ↓
向量相似度搜索 (PGVector)
    ↓
混合检索 (向量 + 全文)
    ↓
返回Top-K相关视频
    ↓
构建上下文注入LLM
    ↓
生成最终答案
```

### 3. 相似度计算

使用余弦相似度：
```
similarity = 1 - (embedding1 <=> embedding2)
```

阈值默认为0.7，可在配置文件中调整。

## 七、性能优化

### 1. 向量索引

数据导入完成后，创建IVFFlat索引：

```sql
CREATE INDEX idx_summary_embedding ON video_vector_store 
    USING ivfflat (summary_embedding vector_cosine_ops) 
    WITH (lists = 100);
```

### 2. 批量处理

推荐使用批量接口处理大量视频，系统会自动分批处理。

### 3. 缓存策略

- 向量计算结果缓存到Redis（默认24小时）
- 避免重复计算相同文本的向量

## 九、常见问题

### Q1: 向量化失败怎么办？

**A**: 检查以下几点：
1. 视频是否已完成分析
2. API Key是否正确
3. 网络连接是否正常
4. 查看日志中的错误信息

### Q2: 如何重新向量化某个视频？

**A**: 使用强制更新参数：
```bash
curl -X POST http://localhost:8080/api/v1/video-vectorization/single/123?forceUpdate=true
```

### Q3: 向量检索结果不准确怎么办？

**A**: 调整以下参数：
1. 降低相似度阈值（similarity-threshold）
2. 增加返回数量（top-k）
3. 优化问题描述

### Q4: 如何处理大量视频？

**A**: 推荐使用异步批量接口：
```bash
curl -X POST http://localhost:8080/api/v1/video-vectorization/batch-async
```
