# BeeEval 视频分析服务设计方案

## 1. 背景与目标

参考现有的Python FastAPI视频分析项目，设计一套适用于BeeEval系统的视频分析服务，实现以下目标：
- 分析存储在OSS中的加密视频
- 结合多模态AI进行智能评估
- 将视频分析结果融入智能问答系统

## 2. 核心差异分析

| 对比项 | Python项目 | BeeEval系统 | 解决方案 |
|--------|-----------|------------|---------|
| 视频存储 | 本地文件 | OSS加密存储 | OSS下载+解密服务 |
| 语音识别 | 本地Whisper | 云端API | 阿里云语音识别/通义千问多模态 |
| 视频处理 | MoviePy | JavaCV/FFmpeg | Java视频处理库 |
| AI模型 | Gemini | 通义千问 | 通义千问多模态API |
| 后端框架 | FastAPI | Spring Boot | 异步任务+线程池 |

## 3. 系统架构设计

### 3.1 整体架构

## 4. 核心服务设计

### 4.1 视频检索服务 (VideoRetrievalService)

**职责**：从OSS获取加密视频并解密

```java
public interface VideoRetrievalService {
    /**
     * 从OSS下载加密视频
     * @param videoId 视频ID
     * @return 本地临时文件路径
     */
    String downloadEncryptedVideo(Long videoId);
    
    /**
     * 解密视频文件
     * @param encryptedFilePath 加密文件路径
     * @return 解密后的文件路径
     */
    String decryptVideo(String encryptedFilePath);
    
    /**
     * 获取视频的OSS签名URL（用于直接访问）
     * @param videoId 视频ID
     * @param expireSeconds 过期时间(秒)
     * @return 签名URL
     */
    String getSignedUrl(Long videoId, int expireSeconds);
}
```

## 5. 通义千问多模态API集成

### 5.1 API调用方式

通义千问支持多模态输入，可以同时传入图像和文本：

```java
// 示例：调用通义千问多模态API
public String callMultimodalApi(String text, String imageUrl) {
    Message userMessage = Message.builder()
        .role(Role.USER.getValue())
        .content(Arrays.asList(
            // 文本内容
            Map.of("type", "text", "text", text),
            // 图像内容（URL或Base64）
            Map.of("type", "image_url", "image_url", Map.of("url", imageUrl))
        ))
        .build();
    
    GenerationParam param = GenerationParam.builder()
        .apiKey(llmConfig.getApiKey())
        .model("qwen-vl-max") // 视觉语言模型
        .messages(List.of(userMessage))
        .build();
    
    Generation gen = new Generation();
    GenerationResult result = gen.call(param);
    
    return result.getOutput().getChoices().get(0).getMessage().getContent();
}
```

## 6. 异步任务处理

### 6.1 使用Spring @Async

```java
@Service
public class VideoAnalysisOrchestrationServiceImpl implements VideoAnalysisOrchestrationService {
    
    @Async("videoAnalysisExecutor")
    @Override
    public CompletableFuture<VideoAnalysisResult> analyzeVideoAsync(Long videoId) {
        // 1. 获取视频信息
        FunctionDomainVideoEntity video = videoService.getById(videoId);
        
        // 2. 下载并解密视频
        String videoPath = videoRetrievalService.downloadEncryptedVideo(videoId);
        String decryptedPath = videoRetrievalService.decryptVideo(videoPath);
        
        // 3. 提取音频和截图
        String audioPath = videoPreprocessService.extractAudio(decryptedPath);
        List<String> frames = videoPreprocessService.captureFrames(decryptedPath, 
            Arrays.asList(0.5)); // 截取中间帧
        
        // 4. 语音转文字
        TranscriptResult transcript = multimodalService.transcribeAudio(audioPath);
        
        // 5. 多模态分析
        VideoAnalysisResult result = multimodalService.analyzeVideo(
            VideoAnalysisRequest.builder()
                .videoId(videoId)
                .transcript(transcript.getText())
                .framePaths(frames)
                .context(buildContext(video))
                .build()
        );
        
        // 6. 保存结果
        resultService.saveResult(result);
        
        // 7. 清理临时文件
        cleanupTempFiles(videoPath, decryptedPath, audioPath, frames);
        
        return CompletableFuture.completedFuture(result);
    }
}
```

### 6.2 线程池配置

```java
@Configuration
@EnableAsync
public class AsyncConfig {
    
    @Bean("videoAnalysisExecutor")
    public Executor videoAnalysisExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(2); // 核心线程数
        executor.setMaxPoolSize(5); // 最大线程数
        executor.setQueueCapacity(100); // 队列容量
        executor.setThreadNamePrefix("video-analysis-");
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        executor.initialize();
        return executor;
    }
}
```

## 9. 技术选型

| 组件 | 技术选型 | 说明 |

## 10. 注意事项

1. **视频加密**: 需要了解现有的加密方式，实现对应的解密逻辑
2. **性能优化**: 视频分析是计算密集型任务，需要合理控制并发数
3. **成本控制**: 通义千问API调用有成本，需要考虑缓存策略
4. **错误处理**: 视频处理可能失败，需要完善的错误处理和重试机制
5. **临时文件**: 及时清理临时文件，避免磁盘空间占用

## 11. 后续扩展

1. **批量预处理**: 对所有视频进行预分析，提升查询响应速度
2. **增量更新**: 只分析新增或修改的视频
3. **自定义评估维度**: 允许用户自定义评估维度和权重
4. **视频对比分析**: 支持多个视频的对比分析
5. **实时分析**: 支持实时视频流的分析
